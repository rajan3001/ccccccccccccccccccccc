did you categorized between prelims and mains exam

Yes — and this separation is extremely important. In your specification you did include it, but it can be improved slightly to make the system clearer and safer.

Right now your schema distinguishes them using two fields:

paperType
questionType

Example:

paperType: Prelims / GS-I / GS-II / GS-III / GS-IV / Essay
questionType: mcq / mains

This works, but in production systems we usually add one more explicit layer to avoid confusion.

1. How Prelims vs Mains Should Be Categorized

Add a dedicated field:

examStage

Example values:

Prelims
Mains
Interview

So your table becomes:

pyq_questions
field	example
examType	UPSC
examStage	Prelims
year	2023
paperType	GS-I
questionType	mcq

Example for mains:

field	example
examType	UPSC
examStage	Mains
paperType	GS-II
questionType	mains
2. Why This Matters

Because the system behavior is different.

Prelims
MCQ
4 options
instant scoring
accuracy metrics
Mains
descriptive answers
AI evaluation
rubric scoring
word limit

If the system depends only on questionType, you can get messy cases.

Example:

questionType: mcq
paperType: GS-I

That could confuse the system.

Explicit stage avoids this.
