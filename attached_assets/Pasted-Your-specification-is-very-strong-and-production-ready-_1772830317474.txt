Without these, ingestion will break frequently.

2. Critical Fix — PDF Extraction Reliability

Right now your ingestion pipeline sends the whole PDF to Gemini.

That is risky.

Some PDFs are:

• scanned images
• multi-column
• include answer keys separately
• contain watermarks

AI will fail on many of them.

Correct Architecture

PDF → Text Extraction → AI Structuring

Not directly:

PDF → AI

Recommended Extraction Pipeline
PDF Upload
   ↓
PDF Text Extraction (pdf-parse / PyMuPDF)
   ↓
If text empty → OCR fallback (tesseract.js)
   ↓
Clean text
   ↓
Send structured text to Gemini
   ↓
AI JSON extraction

This improves accuracy dramatically.

3. AI Prompt Improvements

Your prompt:

Extract every question from this exam paper

Needs stricter formatting rules.

Use something like:

You are extracting questions from an exam paper.

Return ONLY JSON.

Schema:

{
 questionNumber: number,
 questionText: string,
 options: [string,string,string,string] | null,
 questionType: "mcq" | "mains",
 marks: number
}

Rules:
• Preserve original wording
• Do not merge questions
• Do not hallucinate answers
• If unsure about correct answer → null

This prevents hallucinations.

4. Add Validation Layer

Before inserting questions into database.

Add validation step.

Example checks:

questionNumber exists
options length = 4 (for MCQ)
questionText length > 30

Reject invalid entries.

5. Topic Classification Needs a Taxonomy

Right now AI assigns topic freely.

That creates chaos.

Example:

"Indian Constitution"
"Constitution"

"Constitutional Law"

Instead enforce controlled taxonomy.

Example:

AI must choose from predefined list.

This makes trend analysis reliable.

6. Add Duplicate Detection

Very important.

Sometimes PDFs contain:

question
answer key
explanation

which may generate duplicates.

Add dedupe check.

Example:

hash(questionText)

or

examType + year + paperType + questionNumber
