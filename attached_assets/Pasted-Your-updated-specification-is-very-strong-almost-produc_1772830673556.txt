2. Database Design Review

Your schema is very good.

Two minor improvements.

Add index for performance

You will frequently query:

topic

examType

year

Add indexes.

Example:

index(topic)
index(examType, year)
index(textHash)

Without these, trend queries will become slow when the database grows.

Add difficulty field

Later you will want to classify PYQs by difficulty.

Add optional column:

difficulty varchar

Values like:

Easy
Moderate
Hard

AI can classify difficulty during topic classification step.

3. PDF Ingestion Pipeline Improvements

Your ingestion pipeline is correct, but one improvement will drastically increase accuracy.

Instead of sending the entire PDF text to AI, split it first.

Why?

Large PDFs cause LLM extraction errors.

Example pipeline improvement:

PDF text
↓
split by question pattern
↓
AI extraction per block

Example regex:

\d+\.\s

This identifies question boundaries.

Then send smaller chunks to Gemini.

This reduces hallucination risk.

4. OCR Fallback Handling

Better threshold:

500 characters

Never reject the question just because topic classification fails.

Normalize text before hashing.

This avoids false negatives.

9. Trends Prediction Model

Your current prediction logic:

frequency + recency

Good first step.

Improve prediction formula slightly.

Example scoring:

trendScore =
  (frequency * 0.6) +
  (recency * 0.3) +
  (streak * 0.1)

Then classify:

>0.7 → High
0.4–0.7 → Medium
<0.4 → Low

This gives more realistic predictions.
